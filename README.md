# 💍 Сайт-пригласительное на свадьбу

Красивый одностраничный сайт-приглашение на свадьбу с анимациями, фотогалереей и формой-анкетой для сбора данных о гостях.

---

## 📁 Структура проекта

```
invite/
├── index.html    — основная страница (весь контент)
├── style.css     — все стили и анимации
├── script.js     — интерактивность: таймер, форма, анимации
├── api/
│   └── rsvp.js   — Cloudflare Worker: принимает RSVP, пишет CSV в GitHub
├── wrangler.toml — конфиг Cloudflare Worker
└── README.md     — этот файл
```

---

## 🗂 Разделы сайта

| Раздел | Описание |
|---|---|
| **Hero / Главный экран** | Имена, дата, место, кнопка к анкете, летающие лепестки |
| **Таймер обратного отсчёта** | Считает дни / часы / минуты / секунды до свадьбы |
| **Наша история** | Таймлайн ключевых событий пары |
| **Детали торжества** | Карточки с временем и местом церемонии, фуршета, банкета и дресс-кода |
| **Фотогалерея** | Сетка из 6 ячеек с hover-эффектом |
| **Цитаты о любви** | Карусель с авто-переключением |
| **Анкета гостя (RSVP)** | 5 блоков: личные данные, присутствие, еда, трансфер, пожелания |
| **Как нас найти** | Карточка с адресом, метро, парковкой |
| **Footer** | Монограмма, дата, контакты |

## ✏️ Как настроить под свою свадьбу

### 1. Имена и дата

Откройте `index.html` и замените:

```html
<!-- Строка 36-38: имена в Hero -->
<span class="name">Александр</span>
<span class="ampersand">&amp;</span>
<span class="name">Мария</span>

<!-- Строка 45: дата -->
<p class="hero-date fade-in-up delay-3">14 · июня · 2026</p>

<!-- Строка 46: место -->
<p class="hero-location fade-in-up delay-4">Москва · Ресторан «Белый зал»</p>
```

Также обновите `<title>` и `<meta name="description">` в шапке (строки 6–7).

---

### 2. Дата обратного отсчёта

В `script.js` найдите строку:

```js
const weddingDate = new Date('2026-06-14T14:00:00');
```

Замените на вашу дату и время начала церемонии в формате `ГГГГ-ММ-ДД ЧЧ:ММ:СС`.

---

### 3. Наша история (таймлайн)

В `index.html`, раздел `#story` (~строки 80–120). Каждое событие:

```html
<div class="timeline-item reveal">
  <div class="timeline-icon">☕</div>       <!-- emoji-иконка -->
  <div class="timeline-content">
    <span class="timeline-date">Март 2022</span>
    <h3>Первая встреча</h3>
    <p>Описание события...</p>
  </div>
</div>
```

Добавляйте, удаляйте или редактируйте блоки по необходимости.

---

### 4. Детали торжества

Раздел `#details` (~строки 130–175). Каждая карточка:

```html
<div class="detail-card reveal">
  <div class="detail-icon">💐</div>
  <h3>Церемония</h3>
  <p class="detail-time">14:00</p>
  <p>Описание</p>
  <p class="detail-place">Адрес</p>
</div>
```

---

### 5. Фотогалерея

Галерея использует CSS-градиенты как заглушки. Чтобы вставить реальные фото:

В `style.css` найдите классы `.gi-1` … `.gi-6` и замените `background` на путь к фото:

```css
.gi-1 {
  background-image: url('photos/photo1.jpg');
  background-size: cover;
  background-position: center;
}
```

Положите фотографии в папку `photos/` рядом с `index.html`.

---

### 6. Цитаты о любви

Раздел `#wishes` (~строки 196–215). Карточки карусели:

```html
<div class="wish-card">
  <p class="wish-text">«Ваша цитата здесь.»</p>
  <span class="wish-author">— Автор</span>
</div>
```

Добавление новой цитаты требует также добавить новую `<button class="dot">` в блок `#carouselDots`.

---

### 7. Анкета (RSVP)

Форма состоит из 5 блоков:

- **Блок 1** — Имя и фамилия
- **Блок 2** — Сможет присутствовать (да/нет)
- **Блок 3** — Аллергии / непереносимость
- **Блок 4** — Предпочтения по напиткам
- **Блок 5** — Дополнительная информация

Блоки 3–5 показываются только если гость ответил «Да». Чтобы убрать блок — удалите соответствующий `<div class="rsvp-q">`.

---

### 8. Отправка формы (в продакшене)

Данные формы отправляются через **Cloudflare Worker** в CSV-файл в этом же репозитории.

#### Как это работает

1. Гость заполняет форму → `fetch()` к Cloudflare Worker
2. Worker добавляет `submitted_at` (время ответа) и пишет строку в `rsvp.csv` через GitHub API
3. Ты открываешь репозиторий → видишь `rsvp.csv` со всеми ответами

#### Настройка

Подробная инструкция — в разделе **Публикация** ниже.

### 9. Контакты и адрес

В `index.html` найдите и замените:


А также адрес в блоке `#map` и детали карточек в `#details`.

---

### 10. Цветовая схема

В `style.css` все цвета определены через CSS-переменные в `:root {}`. Измените их под свой вкус:

```css
:root {
  --gold:       #c9a96e;   /* основной золотой акцент */
  --gold-light: #e8d5b0;   /* светлый вариант */
  --dark:       #1a1209;   /* тёмный фон */
  --cream:      #fdf8f0;   /* светлый фон секций */
  --text:       #3d2e1a;   /* основной текст */
  --text-light: #7a6a55;   /* второстепенный текст */
}
```

## 🚀 Как запустить

Сайт состоит из чистого HTML/CSS/JS — никакой сборки не нужно.

### Локально (для просмотра)

Просто откройте `index.html` в браузере двойным кликом.

> ⚠️ Шрифты подключаются через Google Fonts — для их загрузки нужен интернет.

### Через локальный сервер (рекомендуется)

Если у вас установлен Node.js:

```bash
npx serve .
# или
npx http-server .
```

Если установлен Python:

```bash
python -m http.server 8080
```

Затем откройте `http://localhost:8080` в браузере.

### Публикация в интернет

Архитектура:
- **GitHub Pages** — хостинг статики (HTML/CSS/JS)
- **Cloudflare Workers** — бэкенд для формы (бесплатно, 100k запросов/день)
- **CSV в репозитории** — все ответы сохраняются в `rsvp.csv`

#### 1. Создайте репозиторий на GitHub

1. Зайдите на [github.com/new](https://github.com/new)
2. Назовите репозиторий, например `invite` (можно любое имя)
3. Создайте репозиторий (без README, .gitignore и лицензии)

#### 2. Залейте код в репозиторий

```bash
# В папке с сайтом:
git init
git add .
git commit -m "init: wedding invitation site"
git remote add origin https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПО.git
git push -u origin main
```

#### 3. Включите GitHub Pages

1. В репозитории: **Settings → Pages**
2. Source: `Deploy from a branch`, Branch: `main`, folder: `/ (root)`
3. Сохраните — через минуту сайт будет доступен по `https://ВАШ_ЛОГИН.github.io/НАЗВАНИЕ_РЕПО/`

#### 4. Создайте GitHub Personal Access Token

1. GitHub: **Settings → Developer settings → Personal access tokens → Fine-grained tokens**
2. Нажмите **Generate new token**
3. Repository access: **Only select repositories** → выберите ваш репозиторий
4. Permissions → Contents: **Read and write**
5. Создайте — скопируйте токен (показывается только один раз)

#### 5. Установите и настройте Cloudflare Worker

1. Зарегистрируйтесь на [cloudflare.com](https://cloudflare.com) (бесплатно)
2. Установите [Node.js](https://nodejs.org)
3. Установите Wrangler CLI:

```bash
npm install -g wrangler
```

4. Войдите в аккаунт Cloudflare:

```bash
wrangler login
```

5. Отредактируйте `wrangler.toml` — замените `your-github-username` на ваш логин GitHub:

```toml
[vars]
GITHUB_OWNER = "ваш-логин"
GITHUB_REPO = "название-репозитория"
GITHUB_BRANCH = "main"
```

6. Установите GitHub токен как секрет:

```bash
wrangler secret put GITHUB_TOKEN
```
Вставьте скопированный токен.

7. Задеплойте Worker:

```bash
wrangler deploy
```
После деплоя вы увидите URL вида `https://wedding-rsvp.ваше-имя.workers.dev`

#### 6. Укажите URL Worker в script.js

Откройте `script.js`, найдите строку:

```js
const RSVP_WORKER_URL = "https://wedding-rsvp.your-username.workers.dev";
```

Замените `your-username` на ваше имя в Cloudflare.

#### 7. Закоммитьте изменения и запушите

```bash
git add .
git commit -m "add Cloudflare Worker for RSVP, update deploy config"
git push
```

Готово! Гости переходят по ссылке GitHub Pages, заполняют анкету — ответы попадают в `rsvp.csv` в корне репозитория.

## ❓ Часто задаваемые вопросы

**Как поменять монограмму «A & M» в навигации?**  
Найдите в `index.html` строку `<div class="nav-logo">A &amp; M</div>` и замените буквы.

**Можно ли добавить музыку на фоне?**  
Да. Добавьте в `index.html` тег `<audio>` и управляйте им через JavaScript. Учтите, что браузеры по умолчанию блокируют автовоспроизведение — лучше добавить кнопку включения.

**Как добавить больше разделов?**  
Скопируйте структуру любого существующего `<section>` и добавьте соответствующий пункт в навигацию.

**Работает ли сайт без интернета?**  
Частично. Без интернета не загрузятся Google Fonts — применятся системные шрифты. Весь остальной функционал работает офлайн.
